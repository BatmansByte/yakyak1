    <div class="section"><h2 class="sectiontitle">Description</h2><p>The <code class="element">abbr</code>
    element is used to provide a fully expanded alternative to an abbreviated
    phrase. The intention is that browsers and other assistive technologies
    will interpret this information and present it to the user in an
    appropriate format when requested.</p>
<p>Some documents, by their very
    nature, may include numerous abbreviations or acronyms. Marking up each
    and every one could cause the document to become very unsightly in some
    browsers—and would be tedious for the author having to mark each one up! A
    general convention is to mark up the first instance of any abbreviation or
    acronym.</p>
</div>


    <div class="example"><div><div><div><h2 class="sectiontitle">Example</h2><p>A simple abbreviation is marked up
    as follows:</p>
<pre class="htmlblock">&lt;p&gt;The page had the phrase <em>&lt;abbr title="Please Turn Over"&gt;PTO&lt;/abbr&gt;</em>
    written on it. If only I knew what PTO meant! I turned the page
    to see if that revealed the answer.&lt;/p&gt;
</pre>
</div></div></div></div>


    <div class="section"><h2 class="sectiontitle">Use This For …</h2><p>The <code class="element">abbr</code>
    element contains the text that’s part of the abbreviation, and has just
    one attribute: the <a title="provides an expanded form of the abbreviation" href="abbr/title"><code class="attribute">title</code></a>
    attribute.</p>
</div>

    </tr></tbody></table><p>Support for
    <code class="element">abbr</code> is varied. Internet Explorer versions 6 and
    earlier don’t understand the element at all, or make it available in the
    DOM tree for interrogation.</p>
<p>In all other browsers, the
    <code class="element">abbr</code> is supported, but in varying visual styles.
    Firefox and Opera underline a marked-up abbreviation that has an
    associated <code class="attribute">title</code> attribute with a light dotted line
    (this can be styled using CSS); Safari and Chrome recognize the
    <code class="element">abbr</code> content, but do not draw attention to it with an
    underline. Firefox, Opera, Chrome and Safari all display the
    <code class="attribute">title</code> attribute for the abbreviation as a tooltip
    (hence, it’s available to mouse users, but not keyboard users). </p>
<p>In
    Firefox, an <code class="element">abbr</code> would display as shown in <a title="" href="#abbr__fig-firefox">Figure 1</a> when moused over.</p>
<div class="fignone" id="abbr__fig-firefox"><span class="figcap">Figure 1. The <code class="attribute">title</code> attribute of
        <code class="element">abbr</code> shown in Firefox</span>
        

        <img src="figures/abbr-example.png" alt="abbr in firefox">
      </div>
<p>In Opera 9.2, the tooltip appears in a similar fashion, but
    with the text “Title:” appended to it, as shown in <a title="" href="#abbr__fig-opera">Figure 2</a>.</p>
<div class="fignone" id="abbr__fig-opera"><span class="figcap">Figure 2. The <code class="attribute">title</code> attribute of
        <code class="element">abbr</code> shown in Opera 9.2</span>
        

        <img src="figures/abbr-opera.png" alt="abbr in opera">
      </div>
<p>In Opera 10, the ‘Title:’ prefix no longer shows, behaving more
    like the Firefox example above.</p>
<p>As a result of the compatibility
    issues associated with it, many people prefer not to use this element.
    Instead, they write it out in full in the document in the first instance,
    as follows:</p>
<pre class="htmlblock">&lt;p&gt;The page had the phrase <em>PTO (Please Turn Over)</em> written on it. If
    only I knew what PTO meant! So I turned the page to see if that
    revealed the answer.&lt;/p&gt;</pre>
<p>The ways in which browsers
    deal with the <code class="element">abbr</code> element—presenting it in a tooltip,
    underlined, italicized, and so on—do little to help users reading the
    content in printed form. This provides yet another reason to simply write
    out the abbreviation in the text, as shown in the HTML above. You may
    consider using generated content in CSS instead, but that approach is
    hindered by some browser support limitations.</p>
<p>Despite variations in
    the presentation of <code class="element">abbr</code> content, the support chart
    shows that full support is provided by all browsers (except for IE 6 and
    earlier), as there are no guidelines in the W3C specifications to indicate
    how browsers should treat such content. If the element is exposed in the
    DOM tree, and if the browser does something with it, that constitutes full
    support.</p>
